1 Purpose

This tutorial is showing you how to deploy a serverless website in AWS the easy way. We will do 3 things in this guide:

  1. Deploy static website with AWS Amplify connected to a git repo
  2. Add a authentication page to access with AWS Cognito
  3. Add a simple example backend with AWS Lambda + API Gateway
Tip

Once you deploy your website instead of adding authentication AND a simple backend you can do one OR the other. I have written these 2 section so that if you don’t want authentication, or you don’t want a backend, you can simply skip the entire section.

2 Setup

In order to deploy a site, we need a site! For this I am going to take the website you are on (isaac-flath.tech) and deploy it behind an authentication page.

See this repository for what I will be deploying exactly, which this website in its current state (before I wrote this post).

2.1 TLDR

  1. Go to AWS Amplify
  2. Scroll down to “Amplify Hosting” and click “Get Started”
  3. Read and follow setup guide
  4. When done go through recommended next steps checklist in AWS console to set up on your own domain, create test sites, add password-protection, PR request previews, etc.

3 Deploy Static Website with Amplify

3.1 Walk Through

Open up AWS Amplify in the AWS console. Once there look for the Amplify Hosting option and click “Get Started”.

We will be going through the setup guide and there’s a few key setting to look out for. For this guide we will be using “Github”, but as you can see below there’s many other options for deploying with Amplify.

On this page you will select the repository with the code. If you have not used Amplify with github before, it will prompt you to authorize it to access your repositories which will require a github login.

Tip

The bottom 2 option allow you to select a branch to deploy from as well as a folder to deploy from. This can be useful if you are using some CI/CD to build your site (such as with nbdev, quarto, etc.)

This next page you will be doing 2 things:

  • App Name: This is an AWS internal name for you to recognize your site. I like to have the source repo it’s connected to in the name.
  • You need to select the checkbox below to allow AWS Amplify to deploy your files.

And you’re done! You will be taken to this page that shows the status of your website. As you can see it’s in the “provision stage” initially, and typically the first deploy takes about 5 minutes.

3.2 Connect to your own URL

At the top you will notice there’s a “Learn how to get the most out of Amplify Hosting” message. This is actually super helpful! Expand it to see links to set up additional features that may be helpful to you. There is a link to Add a Custom Domain, but there’s a pre-requisite in order to do this.

In order to give out website a sensible URL, we first need to register our domain on AWS. We do this through a service called Route 53. Go to the Route 53 resource on AWS and click “Registered Domains” on the left. From there you can choose to register a new domain with AWS, or transfer an existing domain if you already have one.

4 Adding Authentication

The fastest way to put your website behind a login page is to select “Password-protect your site” on the “Learn how to get the most out of Amplify Hosting” section we saw earlier. The downside to this is it’s a password that everyone can use and is not user-specific authentication. If a global password is all you need though, that works just fine!

For this guide we will set up individual user authentication.

4.1 TLDR

  1. Go to AWS Cognito User Pools page
  2. Click create a user pool
  3. Follow setup prompts
